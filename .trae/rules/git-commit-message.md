在此处编写规则，自定义 AI 生成提交信息的风格。
# 整体格式
采用「标题行+可选描述行+可选关联信息」，格式如下：
```
<type>(<scope>): <subject>
// 空行（可选）
<body>
// 空行（可选）
<footer>
```
标题行必填（≤50字符），body可选（每行≤72字符），footer用于关联需求/bug。

# type类型（必选）
type标识操作类型，不可自定义，AI自动匹配，核心类型如下：
|type值|含义|适用场景|
|---|---|---|
|feat|新增功能|新增接口、页面、功能模块|
|fix|修复bug|修复逻辑错误、接口报错等|
|docs|文档修改|修改README、注释等（不涉及代码）|
|refactor|代码重构|优化结构，不新增功能/修复bug|
|其他|style/perf/test等|格式调整、性能优化、测试相关等|

# 核心要求
- scope（可选）：标识影响模块（如user、order、web），多模块可省略。
- subject：动词开头（祈使句），小写无句号，≤50字符，不冗余。
- 禁止项：模糊表述、标题/body过长、自定义type、无关内容、混合修改。

# AI生成提交示例（核心场景）
```
// 新增功能
feat(user): 新增手机验证码登录功能
Closes #102

// 修复bug
fix(order): 修复支付后订单状态未同步问题
Fixes #156

// 文档修改
docs: 更新用户模块接口文档

// 代码重构
refactor(pay): 提取支付公共逻辑
```

# 补充说明
1. 本规范适用于所有提交，团队及AI工具需严格遵循。2. 可根据项目需求补充scope/type，统一同步。3. AI生成后，开发人员需校验确认。
